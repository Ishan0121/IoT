↓

↓

↓

↓
client.stop() → browser renders page → auto-refreshes in 10s

🟢 Simple Solution — Basic Ethernet Web Server

⭐ Start Here — serve a plain-text response. No sensors, no HTML styling needed. Just verify the shield works.
Click Refresh to request page from Arduino...
Simulates a web browser making an HTTP GET request
simple.ino — Exp 09 (Beginner)
// Simple Ethernet Web Server
// W5100 shield: Pins 10-13 reserved for SPI

#include <Ethernet.h>

byte mac[] = { 0xDE, 0xAD, 0xBE, 0xEF, 0xFE, 0xED };
EthernetServer server(80);

void setup() {
  Serial.begin(9600);
  Ethernet.begin(mac);    // Get IP from router via DHCP
  server.begin();
  Serial.print("Server IP: ");
  Serial.println(Ethernet.localIP());
}

void loop() {
  EthernetClient client = server.available();
  if(client) {
    // Wait for blank line (end of HTTP request)
    while(client.connected()) {
      if(client.available()) {
        char c = client.read();
        if(c == '\n') break;
      }
    }
    // Send simple HTTP response
    client.println("HTTP/1.1 200 OK");
    client.println("Content-Type: text/plain");
    client.println();
    client.println("Hello from Arduino!");
    client.println("Uptime: ");
    client.print(millis() / 1000);
    client.println(" seconds");
    client.stop();
  }
}

💾 Advanced Solution — HTML Dashboard + Sensor Data + LED Control

🚀 Advanced — full HTML page, live DHT11 + LDR readings, LED on/off via URL, auto-refresh.
program.ino — Exp 09
#include <Ethernet.h>
#include <DHT.h>

byte mac[]={0xDE,0xAD,0xBE,0xEF,0xFE,0xED};
EthernetServer server(80);

#define DHT_PIN 3
#define DHTTYPE DHT11
DHT dht(DHT_PIN,DHTTYPE);

const int LDR_PIN=A0, LED_PIN=7;
bool ledState=false;

void sendHeaders(EthernetClient& c){
  c.println(F("HTTP/1.1 200 OK"));
  c.println(F("Content-Type: text/html"));
  c.println(F("Connection: close"));
  c.println(); // Blank line = end of headers
}

void sendDashboard(EthernetClient& c,float t,float h,int ldr){
  c.println(F("<!DOCTYPE html><html><head>"));
  c.println(F("<meta http-equiv='refresh' content='10'>"));
  c.println(F("<title>Arduino Dashboard</title>"));
  c.println(F("<style>body{font-family:Arial;background:#1a1a2e;color:#eee;}"));
  c.println(F(".card{background:#16213e;border-radius:12px;padding:20px;margin:12px 0;"));
  c.println(F("border-left:4px solid #00d4ff;}</style></head><body>"));
  c.println(F("<h1 style='color:#00d4ff'>Arduino Dashboard</h1>"));
  c.print(F("<div class='card'>Temperature: <b>"));
  isnan(t)?c.print(F("--")):c.print(t,1); c.println(F(" C</b></div>"));
  c.print(F("<div class='card'>Humidity: <b>"));
  isnan(h)?c.print(F("--")):c.print(h,1); c.println(F(" %</b></div>"));
  c.print(F("<div class='card'>Light: <b>"));
  c.print(ldr); c.println(F(" /1023</b></div>"));
  c.print(F("<div class='card'>Uptime: <b>"));
  c.print(millis()/1000); c.println(F(" s</b></div>"));
  c.println(F("<a href='/led=on'>LED ON</a> <a href='/led=off'>LED OFF</a>"));
  c.println(F("</body></html>"));
}

void setup(){
  Serial.begin(9600); dht.begin();
  pinMode(LED_PIN,OUTPUT); digitalWrite(LED_PIN,LOW);
  if(Ethernet.begin(mac)==0){
    Serial.println(F("DHCP FAILED! Check cable.")); while(true);
  }
  delay(1000); server.begin();
  Serial.print(F("Server at http://")); Serial.println(Ethernet.localIP());
}

void loop(){
  Ethernet.maintain();
  EthernetClient client=server.available();
  if(!client)return;
  String req="";
  unsigned long timeout=millis()+1000;
  bool blank=true;
  while(client.connected()&&millis()<timeout){
    if(client.available()){
      char c=client.read();
      req+=c;
      if(c=='\n'&&blank)break;
      blank=(c=='\n')?true:(c=='\r'?blank:false);
    }
  }
  if(req.indexOf("led=on")>=0){ledState=true;digitalWrite(LED_PIN,HIGH);}
  else if(req.indexOf("led=off")>=0){ledState=false;digitalWrite(LED_PIN,LOW);}
  sendHeaders(client);
  sendDashboard(client,dht.readTemperature(),dht.readHumidity(),analogRead(LDR_PIN));
  delay(1); client.stop();
}

📝 Notes & Extras

💡 W5100 TCP/IP in Hardware
The W5100 chip implements the full TCP/IP network stack in hardware silicon. This offloads all packet assembly, checksums, and retransmission from the Arduino CPU. The ATmega328P only needs to send/receive raw data over SPI — the W5100 handles everything else.
⚠️ DHCP vs Static IP
Ethernet.begin(mac) with just the MAC uses DHCP — router assigns IP automatically. Open Serial Monitor after upload to see the assigned IP. For permanent installations, use Ethernet.begin(mac, staticIP) so the address never changes.
🔧 HTTP Blank Line Rule
HTTP protocol requires a blank line (just \r\n) between headers and body. Without it, browsers won't render the response. The code tracks a blank boolean — when a newline follows a newline, headers are done and we break out of the reading loop.
🌐 Common Shields Reference
Ethernet W5100: wired internet (pins 10–13)
Motor Shield L293D: 2× DC or 1× stepper
GSM Shield: cellular (D2/D3 software serial)
LCD Keypad: 16×2 display + 5 buttons (A0, 4–8)
Relay Shield: 4× relays (pins 4–7)

🔧 Key Functions

Ethernet.begin(mac)
Initialize Ethernet with DHCP. Returns 0 on failure. Call once in setup(). The MAC address must be unique on your LAN.
Ethernet.maintain()
Must be called in loop() to renew DHCP lease before it expires. Keeps the IP address valid for long-running sketches.
server.available()
Returns an EthernetClient if a browser has connected. Returns null/falsy if no client. Check at every loop() iteration.
client.println(F("HTTP/1.1 200 OK"))
Send HTTP response line to connected browser. Use F() macro to store strings in Flash and save SRAM — critical in a web server sketch with many string constants.
client.stop()
Close the connection after sending the response. HTTP/1.0 requires explicit close. Without this, browser waits for more data and dashboard appears blank.

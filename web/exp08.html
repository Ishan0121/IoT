Experiment 08

Spinning a DC Motor &
Motor Speed Control

Control DC motor speed and direction with L298N H-bridge. Pot-based speed control, soft-start ramping, and debounced direction toggle.

🎯 L298N · H-Bridge · PWM · Debounce Intermediate ENA=Pin9 · IN1=5 · IN2=6

🔌 L298N H-Bridge Circuit

⚡ L298N Connections
  • IN1 → Pin 5 (direction A)
  • IN2 → Pin 6 (direction B)
  • ENA → Pin 9 (PWM speed)
  • 12V → External 9–12V supply
  • GND → L298N GND + Arduino GND
  • OUT1/OUT2 → Motor terminals
🔄 Direction Logic
  • FORWARD: IN1=HIGH, IN2=LOW
  • REVERSE: IN1=LOW, IN2=HIGH
  • BRAKE: IN1=HIGH, IN2=HIGH
  • COAST: IN1=LOW, IN2=LOW
  • Speed: analogWrite(ENA, 0–255)
🛡️ Safety Rules
  • Brief BRAKE before reversing direction
  • Link L298N GND to Arduino GND!
  • Dead-band: <10% PWM → stop cleanly
  • Soft-start ramp: ±3 per loop iteration
  • Debounce button 50ms before acting

🟢 Simple Solution — Basic DC Motor Speed Control

⭐ Start Here — spin motor forward, then reverse. Fixed speed, no potentiometer or buttons needed.
Current State
FORWARD (PWM: 128)
Simulates basic delay() sequential motor control
simple.ino — Exp 08 (Beginner)
// Simple DC Motor Control with L298N
// IN1=5, IN2=6, ENA=9(PWM)

const int IN1=5, IN2=6, ENA=9;

void setup() {
  pinMode(IN1, OUTPUT);
  pinMode(IN2, OUTPUT);
  pinMode(ENA, OUTPUT);
}

void loop() {
  // Forward at half speed (PWM = 128 / 255)
  digitalWrite(IN1, HIGH); digitalWrite(IN2, LOW);
  analogWrite(ENA, 128);
  delay(3000);

  // Stop
  analogWrite(ENA, 0);
  delay(500);

  // Reverse at full speed (PWM = 255)
  digitalWrite(IN1, LOW); digitalWrite(IN2, HIGH);
  analogWrite(ENA, 255);
  delay(3000);

  // Stop
  analogWrite(ENA, 0);
  delay(500);
}

💾 Advanced Solution — Pot Speed + Soft-Start + Debounced Button

🚀 Advanced — potentiometer speed control, soft-start ramp, debounced direction toggle button.
program.ino — Exp 08
// Exp 08 — DC Motor Speed & Direction Control
// L298N: IN1=5, IN2=6, ENA=9(PWM)
// Pot=A0, Direction button=2 (INPUT_PULLUP)

const int IN1=5, IN2=6, ENA=9, POT=A0, BTN=2;

bool motorForward=true;
bool lastBtnState=HIGH;
int  currentPWM=0;

void setDirection(bool fwd){
  digitalWrite(IN1, fwd?HIGH:LOW);
  digitalWrite(IN2, fwd?LOW:HIGH);
}

// Soft-start: ramp ±3 per call
void rampTo(int target){
  if(currentPWM<target) currentPWM=min(currentPWM+3,target);
  else if(currentPWM>target) currentPWM=max(currentPWM-3,target);
  analogWrite(ENA, currentPWM);
}

void brakeMotor(){
  digitalWrite(IN1,HIGH); digitalWrite(IN2,HIGH);
  analogWrite(ENA,0); currentPWM=0;
}

// Debounced direction toggle
void checkButton(){
  bool btnNow=digitalRead(BTN);
  if(lastBtnState==HIGH&&btnNow==LOW){
    delay(50);
    if(digitalRead(BTN)==LOW){
      brakeMotor(); delay(200);
      motorForward=!motorForward;
      setDirection(motorForward);
      Serial.println(motorForward?F("→ FORWARD"):F("← REVERSE"));
    }
  }
  lastBtnState=btnNow;
}

void setup(){
  Serial.begin(9600);
  pinMode(IN1,OUTPUT); pinMode(IN2,OUTPUT);
  pinMode(ENA,OUTPUT); pinMode(BTN,INPUT_PULLUP);
  brakeMotor(); setDirection(true);
  Serial.println(F("=== DC Motor Control ==="));
}

void loop(){
  int potVal=analogRead(POT);
  int target=map(potVal,0,1023,0,255);
  if(target<25) target=0; // dead-band <10%
  rampTo(target);
  if(target>0) setDirection(motorForward);
  checkButton();
  delay(20);
}

📝 Notes & Extras

💡 H-Bridge Explained
An H-Bridge has 4 switches (transistors) arranged like the letter H. By closing different switch pairs, current flows through the motor in either direction. L298N contains two H-bridges (Channel A + B), allowing control of 2 motors or 1 stepper motor simultaneously.
⚠️ Shared GND is Critical
The external motor supply (9–12V) and Arduino must share a common GND. Without a common ground, the control signals from Arduino pins have no reference point and the L298N won't respond correctly. Always link GNDs before powering up.
🔧 Soft-Start Ramping
Jumping directly to full PWM causes high inrush current that stresses motor windings and can reset Arduino. rampTo() increments by 3 each loop (every 20ms) — reaching full speed in ~1.7 seconds. This protects the motor and ensures smooth acceleration.
🔘 Button Debounce
Mechanical buttons bounce 5–50ms when pressed. Without debounce, one press registers 10–20 events. Solution: detect falling edge (HIGH→LOW), then delay(50ms), then re-check if still LOW. Only then act. This is called software debounce.

🔧 Key Functions

analogWrite(ENA, 0–255)
PWM on Enable pin controls motor speed. 0=stopped, 255=full speed. Works on PWM-capable pins only (D3,D5,D6,D9,D10,D11).
rampTo(target)
Custom soft-start: gradually changes currentPWM toward target by ±3 each call. Prevents current spikes and mechanical stress.
brakeMotor()
Sets IN1=IN2=HIGH and ENA=0. Shorts motor terminals — creates electromagnetic braking (fast stop). Call before direction reversal.
checkButton()
Edge-detect + debounce pattern: detect HIGH→LOW transition, wait 50ms, verify still LOW, then toggle direction with a brake buffer.
map(val, iMin, iMax, oMin, oMax)
Scale pot reading 0–1023 to PWM 0–255. Then apply dead-band: values below 10% (25/255) become 0 for clean motor stop.

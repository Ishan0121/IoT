Experiment 03

Blinking LED Sketch
with Arduino

Control an LED with delay-based and non-blocking millis() timing. Understand PWM fade using analogWrite on PWM-capable pins.

🎯 Objective: LED Control & Timing Beginner Pin 9 (PWM)

🔌 Circuit Diagram

🧩 Components
  • Arduino UNO
  • 1× LED (any color)
  • 1× 220Ω resistor
  • Breadboard + jumper wires
🔌 Wiring
  • Pin 9 → 220Ω → LED(+) anode
  • LED(−) cathode → GND
  • Pin 9 is PWM-capable
  • 220Ω limits current to ~14mA
⚠️ Why 220Ω?
  • LED forward voltage ~2V
  • (5V − 2V) / 0.014A = 214Ω
  • 220Ω is the nearest standard value
  • Max pin current: 40mA — don't exceed!

🟢 Simple Solution — delay() Blink

⭐ Start Here — uses only delay() and digitalWrite(). Perfect for your first sketch.

simple.ino — Exp 03 (Beginner)
// Simple LED Blink — the classic first sketch!
// Circuit: built-in LED on Pin 13 (no extra components needed)

const int LED_PIN = 13;  // Pin 13 has a built-in LED on Arduino UNO

void setup() {
  pinMode(LED_PIN, OUTPUT);  // Set pin as output
}

void loop() {
  digitalWrite(LED_PIN, HIGH);  // Turn LED ON
  delay(1000);                  // Wait 1 second
  digitalWrite(LED_PIN, LOW);   // Turn LED OFF
  delay(1000);                  // Wait 1 second
}                                // (repeat forever)

💾 Advanced Solution — millis() + PWM Fade

🚀 Advanced — non-blocking timing with millis() and PWM brightness fade.
program.ino — Exp 03 
/*
 * Experiment 03 — Blinking LED Sketch
 * Non-blocking blink + PWM fade demonstration
 * Circuit: Pin 9 → 220Ω → LED → GND
 */

const int LED_PIN  = 9;     // PWM-capable pin
const int ON_TIME   = 700;   // ms LED stays ON
const int OFF_TIME  = 300;   // ms LED stays OFF

int           blinkCount = 0;
bool          ledState   = false;
unsigned long lastToggle = 0;

void setup() {
  Serial.begin(9600);
  pinMode(LED_PIN, OUTPUT);
  digitalWrite(LED_PIN, LOW);
  Serial.println(F("=== LED Blink Experiment ==="));
}

// ── Non-blocking blink using millis() ──────
void updateBlink() {
  unsigned long now      = millis();
  unsigned long interval = ledState ? ON_TIME : OFF_TIME;

  if (now - lastToggle >= interval) {
    lastToggle = now;
    ledState   = !ledState;
    digitalWrite(LED_PIN, ledState ? HIGH : LOW);
    if (ledState) {
      blinkCount++;
      Serial.print(F("Blink #")); Serial.println(blinkCount);
    }
  }
}

// ── PWM Fade (called every 10 blinks) ──────
void fadeLED(int from, int to, int stepDelay) {
  int step = (from < to) ? 1 : -1;
  for (int b = from; b != to + step; b += step) {
    analogWrite(LED_PIN, b);
    delay(stepDelay);
  }
}

void loop() {
  updateBlink();

  // Every 10 blinks, run a PWM fade sequence
  if (blinkCount > 0 && blinkCount % 10 == 0) {
    static int lastFade = 0;
    if (blinkCount != lastFade) {
      lastFade = blinkCount;
      Serial.println(F("--- PWM Fade ---"));
      fadeLED(0, 255, 5);   // Fade in
      fadeLED(255, 0, 5);   // Fade out
      lastToggle = millis();  // Re-sync timer
    }
  }
}

📝 Notes, Extras & Key Concepts

⚠️ Why delay() is Bad
delay(1000) freezes the entire Arduino for 1 second — it can't read buttons, update displays, or react to anything. This is called blocking. Use millis() to achieve the same timing without blocking: check if enough time has passed each loop iteration.
💡 PWM — Pulse Width Modulation
Digital pins can only be HIGH or LOW. PWM simulates analog voltage by rapidly switching between HIGH and LOW. The ratio of time spent HIGH vs LOW is called the duty cycle. analogWrite(pin, 127) → 50% duty → LED appears at half brightness.
🔋 millis() Overflow
millis() returns an unsigned long that overflows back to 0 after ~49.7 days. Using subtraction (now - lastToggle) handles this automatically and correctly — the math wraps safely because both variables are unsigned.
📐 resistor Calculation
R = (Vsupply − Vforward) / I_desired
= (5V − 2V) / 0.015A = 200Ω
Use 220Ω (nearest standard value). For 3.3V supply: (3.3V − 2V) / 0.01A = 130Ω → use 150Ω. Always limit LED current to 10–20mA max.

🔧 Key Functions

digitalWrite(pin, value)
Set digital pin to HIGH (5V) or LOW (0V). Pin must be set as OUTPUT with pinMode() first. Cannot control brightness — only on or off.
analogWrite(pin, value)
Output a PWM signal. Value 0–255 (0% to 100% duty cycle). Only works on PWM pins: D3, D5, D6, D9, D10, D11 on UNO. Does NOT output a true analog voltage.
millis()
Returns milliseconds since program start as unsigned long. Use for non-blocking timing: if (millis() - lastTime >= interval) { ... }.
delay(ms)
Pause execution for specified milliseconds. Blocks ALL other processing including interrupt-driven tasks. Avoid for anything beyond simple demos.

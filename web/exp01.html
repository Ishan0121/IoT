📌 Pin Types Explained

⚡ Digital Pins (D0–D13)
  • Output: HIGH=5V, LOW=0V
  • D0, D1 = UART RX/TX — avoid for sensors during debug
  • PWM pins (marked ~): D3, D5, D6, D9, D10, D11
  • analogWrite(pin, 0–255) → duty cycle control
📊 Analog Pins (A0–A5)
  • 10-bit ADC → reads 0 to 1023
  • Maps: 0V→0, 5V→1023
  • Can also function as digital pins
  • A4 = SDA, A5 = SCL for I2C bus
🔋 Power Pins
  • 5V — regulated 5V output (max 200mA)
  • 3.3V — regulated 3.3V (max 50mA)
  • GND — ground (two pins)
  • Vin — unregulated from barrel jack
  • RESET — pull LOW to restart

🟢 Simple Solution — Minimal Board Test

⭐ Start Here — just 2 functions to confirm the board is alive: blink LED + print to Serial Monitor.
simple.ino — Exp 01 (Beginner)
// Minimal Arduino UNO Board Test
// Tests: LED, Serial, and one analog pin

void setup() {
  Serial.begin(9600);            // Start Serial Monitor
  pinMode(13, OUTPUT);          // Built-in LED on Pin 13
  Serial.println("Arduino UNO ready!");
}

void loop() {
  digitalWrite(13, HIGH);         // LED ON
  Serial.print("A0 reading: ");
  Serial.println(analogRead(A0));  // Print analog value 0–1023
  delay(500);
  digitalWrite(13, LOW);          // LED OFF
  delay(500);
}

💾 Advanced Solution — EEPROM Counter + millis() Timing

🚀 Advanced — non-blocking millis() blink, EEPROM reset counter, multi-channel analog reporting.
program.ino — Exp 01 
/*
 * Experiment 01 — Understanding Arduino UNO Board
 * Demonstrates: Serial report, LED blink, Analog reads, EEPROM counter
 */

#include <EEPROM.h>

const int LED_PIN       = 13;   // Built-in LED
const int BAUD_RATE     = 9600;
const int BLINK_INTERVAL = 500;  // ms

int           loopCounter  = 0;
bool          ledState     = false;
unsigned long lastBlinkMs  = 0;
unsigned long lastPrintMs  = 0;

void setup() {
  Serial.begin(BAUD_RATE);
  pinMode(LED_PIN, OUTPUT);

  // EEPROM reset counter
  int resetCount = EEPROM.read(0);
  if (resetCount == 255) resetCount = 0;
  EEPROM.write(0, ++resetCount);

  delay(200);
  Serial.println(F("=== Arduino UNO Board Report ==="));
  Serial.println(F("CPU  : ATmega328P @ 16 MHz"));
  Serial.println(F("Flash: 32KB | SRAM: 2KB | EEPROM: 1KB"));
  Serial.print(F("Reset #: ")); Serial.println(resetCount);
}

void loop() {
  unsigned long now = millis();

  // Non-blocking LED blink
  if (now - lastBlinkMs >= BLINK_INTERVAL) {
    lastBlinkMs = now;
    ledState = !ledState;
    digitalWrite(LED_PIN, ledState ? HIGH : LOW);
  }

  // Print status every 1 second
  if (now - lastPrintMs >= 1000) {
    lastPrintMs = now;
    loopCounter++;
    Serial.print(F("Loop #")); Serial.print(loopCounter);
    Serial.print(F(" | A0:")); Serial.print(analogRead(A0));
    Serial.print(F(" A1:"));  Serial.print(analogRead(A1));
    Serial.print(F(" | Uptime:")); Serial.print(now/1000); Serial.println(F("s"));
  }
}

📝 Notes, Extras & Gotchas

💡 Why use millis() not delay()?
delay() blocks the CPU completely — nothing else runs. millis() returns the time elapsed since boot, letting you check "has enough time passed?" without freezing. This is called non-blocking timing and is essential for any sketch that does multiple things simultaneously.
⚠️ Floating Analog Pin
Leave A0 unconnected (floating) and analogRead(A0) returns random garbage between 0–1023. Always tie an analog input to GND or VCC through a pull-down/pull-up resistor when not using a sensor — otherwise electromagnetic noise produces unpredictable values.
🔋 EEPROM Write Limits
ATmega328P EEPROM is rated for ~100,000 write cycles per address. Writing in a tight loop will destroy it quickly. Use EEPROM.update(addr, val) instead of EEPROM.write() — it checks if the value changed first, skipping unnecessary writes.
📡 D0/D1 Conflict
D0 and D1 are shared with the USB-Serial converter. When Serial.begin() is active, do NOT connect sensors to D0 or D1 — the serial traffic will corrupt your sensor readings and vice versa.

🔧 Key Functions & APIs

pinMode(pin, mode)
Set a pin as INPUT, OUTPUT, or INPUT_PULLUP. Must be called in setup() before using the pin.
digitalWrite(pin, value)
Write HIGH (5V) or LOW (0V) to a digital pin configured as OUTPUT.
analogRead(pin)
Read analog voltage on A0–A5. Returns 0–1023 mapping 0V–5V via 10-bit ADC. Takes ~100μs.
millis()
Returns milliseconds since the sketch started running. unsigned long. Overflows after ~49.7 days (handled gracefully with subtraction).
Serial.begin(baud)
Initialize UART at specified baud rate. Use 9600 for most sketches. Must match Serial Monitor setting.
EEPROM.read(addr) / EEPROM.write(addr, val)
Read/write a byte (0–255) at an EEPROM address (0–1023 on UNO). Persists across power cycles.
